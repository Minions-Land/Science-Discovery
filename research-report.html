为什么把"调研"和"proposal"合并成一份

做学术 AI 的研究不能凭直觉——你要先清楚 field 现在有什么（别人做了什么、指标怎么算、我们在哪个坐标位），再决定我们做什么（方法、资源、时间）。 这份文档把两件事缝在一起：

  • Part I · 调研（§1–§5） 是技术性 survey：本组 7 个仓库 → 社区完整生态 → 各环节 benchmark → 2026 最新 arXiv 关注点 → 网络通路。这一半读完能独立输出。
  • Part II · Proposal（§6–§15） 是给学生的招募与工作说明：背景 → 假设 → track → 环境脚本 → 路线图 → 评价标准。这一半告诉你加入后具体怎么做。

两边共享同一套数据：GitHub star 来自 api.github.com 实时；benchmark 标题/年份/DOI 来自 api.openalex.org；本地仓库清单来自 /Users/mjm/Skill 实地遍历。生成日期 2026-05-11。

4 · 前沿关注 · Agent-Native Research Artifact (ARA)

arXiv 2604.24658 · Liu et al. 2026-04 · 这个方向最值得盯的一篇

arXiv 2604.24658 · 2026-04-27 38 位作者（MIT Media Lab / Michigan / Stanford …） 代码 Orchestra-Research/Agent-Native-Research-Artifact · 124 ⭐ MIT License 强烈推荐

这篇论文的标题叫 "The Last Human-Written Paper: Agent-Native Research Artifacts"， 是 2026 年我见过对"论文这个对象本身"最激进的重构提议。和本 proposal 高度共振——它给出了一个协议 + 三个 skill，直接在 PaperBench 和 RE-Bench 上把复现表现拉高。

4.1 动机：两种"税"

作者指出当前论文格式对 AI agent 有两种结构性成本：

  • Storytelling Tax（叙事税） — 真实研究是分叉的探索过程，发表时被压扁成线性叙事，失败实验、被拒假设、替代方案全被丢弃。
  • Engineering Tax（工程税） — "够 reviewer 读的文字" 和 "够 agent 执行的规格" 之间有鸿沟，关键实现细节没写出来。
两个震撼的量化数字
· 23 篇 ICML 2024 论文里只有 45.4% 的复现需求（8,921 条）在 PDF 里完整指定。
· RE-Bench 上 24,008 次 agent 运行，失败运行占 90.2% 的总美元成本——没有 prior failure 记录时每个 agent 都在独立重走死胡同。

4.2 ARA 协议：论文的四层结构

ARA 把论文从叙事文档替换成机器可执行的知识包，按四层组织：

层目录装什么
Logic（认知层）
What & Why		logic/problem.md · claims.md（可证伪断言）· concepts.md · experiments.md · solution/{architecture,algorithm,constraints,heuristics}.md · related_work.md
Physical（物理层）
How		src/configs/（超参 + 理由）· environment.md（依赖、硬件、seed）· 完整可执行代码
Exploration（探索图）
Journey		trace/exploration_tree.yaml —— 研究 DAG，带类型节点 + dead ends
Evidence（证据）
Raw proof		evidence/tables/（精确结果表）· figures/（提取的数据点）

顶层 PAPER.md 是 ~200 token 的 manifest，告诉 agent 这个 artifact 是否值得继续读——这是"progressive disclosure"。 跨层有 forensic binding：claim 引用 experiment，experiment 引用 evidence，heuristic 引用 code。 每一项带 provenance tag（user / ai-suggested / ai-executed / user-revised），区分人类确认 vs AI 推断。

11 · 怎么"打"一个 benchmark

方法论 checklist，每个 track 都会用到

  1. 锁定主榜 — 按能力选。Deep research → GAIA + BrowseComp；ML → MLE-Bench + RE-Bench；端到端 → PaperBench + AAAR-1.0。
  2. 挂一个辅榜 — 证明没过拟合。主 GAIA 就配 AssistantBench；主 MLE-Bench 就配 ScienceAgentBench。
  3. 对齐 4 条受控变量（评审一定会问）：
    • 底座模型（Claude Sonnet 4.6 vs GPT-5，哪个都行但全程固定一个）
    • 工具预算（浏览次数 / 代码执行次数 / max tokens）
    • 时钟 + 成本预算（wall-clock 秒 / USD per task）
    • 评委（LLM-judge 固定一个 GPT-4o；重要结论叠一层人类）
  4. 报告五条标配指标：task success / cost per task / wall-clock / citation faithfulness / human preference
  5. 失败模式分桶（2025 后事实标准）： ToolError Hallucination Plan-break Format-drift
  6. 公开 trajectory — 代码 + 每题 JSONL + rubric 脚本。不放 trajectory 的 agent 论文容易被拒。
ARA 的额外建议
如果走 Track 04，把你方法的结果以 ARA artifact 形式发布（/compiler 自动编译）——这样后续 agent 复现你的工作时能直接读 trace/ 和 evidence/，相当于免费加一份可复现性证据。

13 · 评价标准

学生项目"做得好不好"的内部打分表，完全透明

维度权重满分是什么样
可复现性30%一个完全不懂的人按 README 能一键重跑主实验，拿到 ±3% 以内的数字
实验严谨25%≥ 3 seed；受控变量表；辅榜；失败模式分桶
技术贡献20%有 baseline 之外的 insight 或方法，能说清"为什么别人没做"
写作质量15%短、结构清、图表自洽、引用完整
协作过程10%定期 commit、周会主动发言、文档记录痕迹清晰
我们更在意过程而不是结果
benchmark 没打过 baseline 不会导致项目失败。你的 trajectory 和失败分析本身就是贡献——2025 年起多个 agent 论文接收的核心卖点就是"系统性的失败模式分类"。ARA 论文的核心动机之一也正是这个。

14 · 常见问题

历年带学生攒下来的那些问题

我是大二，只上过 Python 基础课，能加入吗？
能。Track 01（Skill 拉取与基准建立）就是按这个起点设计的。主要是 Git/Bash/Markdown，把仓库跑通、截图、写一页说明。做完一轮就可以升到 Track 02。
没有 Claude API 账号怎么办？
Claude Code 本地免费能跑 skill 结构；真正跑 benchmark 时我们会发共用 API key（受控预算）。开源模型路线（DeepSeek / Qwen）也有 Track，跑本地推理。
只做 Track 01 能写进简历吗？
能，但角度要选对。写法不是"我拉了 10 个仓库"，而是"我维护了 X 个学术 agent skill 的首个中文 benchmark 卡片，覆盖 N 个研究环节"——关键词是维护者 maintainer 不是 user。
ARA 那套协议看起来很新，学生能上手吗？
能。代码在 MIT 许可的公开 repo，一条 npx @orchestra-research/ara-skills 装完。起点最低的做法：用 /compiler 把你自己之前写过的任何一篇作业 / 小论文 / 课程 project 编译成 ARA artifact 看看效果，这个练手成本很低。
如果我的实验结果比官方 paper 差很多，算失败吗？
不算。三种情况都常见：(a) paper 报的是 cherry-picked run，你遇到 variance；(b) paper 的 prompt 没完全公开；(c) 底座模型版本变了。把差距可信地量化出来本身就是结论。
为什么要投 workshop 而不是主会？
时间窗对得上。主会 deadline 间隔 6 个月，workshop 1–2 个月一轮。学生第一篇先 workshop，拿到 reviewer 反馈后再升主会是成功率最高的路径。
中文学术 benchmark（Track 03）真的算贡献吗？
算。所有主流榜单（GAIA / PaperBench / AutoSurvey / SimpleQA）都只评英文，中文侧有真实空缺。关键是 reference 要硬：用 CNKI/万方的真实引用池做金标，不是 GPT 生成的伪引用。
周会是什么节奏？
